The Simplest Solution for Intranet Penetration Without a Public IP

Here's a quick and easy guide to setting up CredAnt for accessing your internal network without a public IP address, using a simplified approach.

Server-Side Installation (Example: Ubuntu 20.04)

  1. 1. Download and Install:Run the following command in your server's terminal. This will download (about 23MB) and install CredAnt:
    curl -s https://credata.net/cred_ant/install.sh | bash
  2. 2. Initialization:Initialize CredAnt with root privileges (using sudo if necessary):
    sudo cred_ant init
    This command generates a Node ID. Copy this ID; you'll need it later.
  3. 3. Obtain a License:
    • • Join the Telegram group: https://t.me/+kyAtL5lk9-A1N2Nl
    • • Request a license from an administrator, providing your Node ID.
    • • The administrator will provide you with a license string.
    • • Paste the license string into the initialization prompt. Use default options for other prompts.
  4. 4. Start and Enable CredAnt:After successfully importing the license, run these three commands:
    systemctl start cred_ant
    systemctl enable cred_ant
    systemctl status cred_ant
    Verify that the status is active.
  5. 5. Verify IP Address:Check for a new IP address starting with 10.18.1 using:
    ifconfig

macOS Installation

  1. 1. Download and Extract:Download the appropriate macOS package from the website and extract it in your Downloads directory. (This example uses the amd64 version; adjust as needed for M1/M2 chips).
    cd Downloads  # Navigate to the Downloads directory
    curl -o cred_ant_0.1.14.zip https://uglyfirefish.github.io/credant/downloads/cred_ant_macos-amd64_v0.1.14.zip # Download the package
    unzip cred_ant_0.1.14.zip  # Unzip the package
    cd darwin/amd64/        # Navigate into the extracted directory
  2. 2. Copy to /usr/local/bin/ (Important):Copy the cred_ant executable to /usr/local/bin/ so you can run it from any directory in the terminal.
      cp cred_ant /usr/local/bin/
  3. 3. Initialization:Run the initialization command with root privileges:
    sudo -i cred_ant init
    This command generates a Node ID. Copy this ID; you'll need it later.
  4. 4. Obtain a License:
    • • Join the Telegram group: https://t.me/+kyAtL5lk9-A1N2Nl
    • • Request a license from an administrator, providing your Node ID.
    • • The administrator will provide you with a license string.
    • • Paste the license string into the initialization prompt. Use default options for other prompts.
  5. 5. Start CredAnt:Start CredAnt:
    sudo -i cred_ant start

Keep this terminal window open. It will display ongoing information.

  1. 6. Verify IP Address (New Terminal Window):Open a new terminal window and check for a new IP address starting with 10.18.1:
    ifconfig
  2. 7. Initial Ping Test:
    From the mac, ping the server's virtual IP (e.g., 10.18.1.1):
     ping 10.18.1.1

Windows Installation

  1. 1. Download and Extract:Download the Windows program from the website to your Downloads directory. It's recommended to extract the files to C:\credant.
  2. 2. Open PowerShell as Administrator:
    • • Right-click the Start Menu.
    • • Choose "Windows PowerShell (Admin)" or "Terminal (Admin)".
  3. 3. Navigate to the Directory:
    cd C:\credant  # Or the directory where you extracted CredAnt
  4. 4. Initialization:Run the initialization command:
    .\cred_ant.exe init
    This command generates a Node ID. Copy this ID; you'll need it later.
  5. 5. Obtain a License:
    • • Join the Telegram group: https://t.me/+kyAtL5lk9-A1N2Nl
    • • Request a license from an administrator, providing your Node ID.
    • • The administrator will provide you with a license string.
    • • Paste the license string into the initialization prompt. Use default options for other prompts.
  6. 6. Start CredAnt:
     .\cred_ant.exe start
  7. 7. Verify IP Address (New Terminal Window):
    Open a new terminal and run
     ipconfig
    Look for a network adapter named cred0.
